Перечислите известные вам инструменты и приёмы для отладки кода в разных браузерах.

Я думаю, всё начиналось с alert(). Ковыряние в исходном коде, сотни всплывающих окошек + можно ещё и забыть за собой удалить. Не самый удобный способ, мне кажется =)

Затем пришёл console.log(). Уже более современная практика, и по сей день используется в проектах в качестве client-side логгирования. В IE console бывает undefined, так что следует проверять заранее.

Теперь почти во всех современных браузерах есть очень эффективные Developer Tools, которые только тосты не жарят. Обычно они включатся нажатием F12. Кроме всего прочего, в дебаггер можно попасть используя ключевое слово debugger.

  • Internet Explorer 8/9

    Лишний раз я стараюсь не встречаться с инструментами разработчика для пользователей IE и писать более-менее кроссбраузерный JS/CSS, но иногда приходится отлаживать упавший код. Несмотря на не самое фантастическое удобство, функций для отладки в IE достаточно.

  • Google Chrome

    На мой взгляд, у браузера от Google на данный момент самые продвинутые и быстро развивающиеся инструменты. Экстремально удобная отладка кода, установка брейкпоинтов на события, ajax запросы, профилирование кода и сети, манипуляция DOM, автокомплит по функциям, свойствам, селекторам. В общем, одно удовольствие.

    У Оперы так же есть встроенные инструменты, Dragonfly, но ими я никогда не пользовался.

  • Firefox

    FireBug не устанавливается вместе с FireFox, идёт как отдельное расширение. Во многом повторяет функциональность Chrome Dev Tools. Шучу. Наоборот, конечно же. Хотя мне кажется, что некоторое преимущество у хрома есть. Дело привычки.

Конечно же, лучше всего тот код, который отлаживать не нужно. Писать юнит тесты, благо в JavaScript сейчас уже достаточно развита эта область, плюс интеграционные тесты с помощью фреймворков вроде Selenium.

Задание 2

Опишите своими словами, что происходит в браузерах при клике на ссылку yandex.ru (полнота ответа — на ваше усмотрение).

  1. Генерируется событие наведения мыши mouseover
  2. Применяется CSS псевдокласс hover
  3. Генерируется событие нажатия mousedown
  4. Применяется CSS псевдокласс active
  5. Если мышка убрана с элемента, то это расценивается как drag&drop, клик не засчитан
  6. Иначе генерируется событие mouseup и, собственно, click
  7. Если событие по умолчанию для ссылок не прервано (e.preventDefault() или return false), то браузер совершает переход.

Касательно генерации событий: событие создаётся и спускается вниз, последовательно проходя все родительские элементы до элемента-цели. Эта фаза называется capture фазой. Затем, событие поднимается обратно вверх. Эта фаза называется bubble фазой. Можно отменить распространение события методом e.stopPropogation().

Обработчики событий вызываются в том порядке, в котором были добавлены.
